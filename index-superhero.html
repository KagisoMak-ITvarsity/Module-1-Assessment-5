üåê Internet Principles Covered

üì± Client-Server

Understand how clients request data and servers respond with information.

  • Request-Response Model
  • Client Processing
  • Server Responses
üìñ Learn More

How it works:

Browser (client) sends a request ‚Üí Server receives and processes it ‚Üí Server sends response with data ‚Üí Browser receives and displays the information

Real World: When you visit a website, your browser requests the HTML, CSS, and images from a server. The server responds with those files, and your browser displays them.

üó∫Ô∏è IP Routing

How data packets find their way across networks using unique IP addresses.

  • IP Addressing
  • Network Routing
  • Data Packets
üìñ Learn More

How it works:

Every device has a unique IP address (like 203.0.113.45). Data is broken into packets, each marked with source and destination IP. Routers forward packets through the best path to reach the destination.

Real World: Like mailing letters - each has an address, and the postal system routes them through various facilities to reach the correct destination.

ü§ù TCP/IP Protocol

The backbone protocol suite ensuring reliable internet communication.

  • Reliable Delivery
  • Ordered Transmission
  • Error Handling
üìñ Learn More

How it works:

TCP (Transmission Control Protocol): Ensures packets arrive in the correct order and handles retransmission if packets are lost.

IP (Internet Protocol): Handles addressing and routing of packets.

Result: Together they provide reliable, ordered delivery of data across the internet.

üìñ Domain Name System

Translating human-readable domain names to IP addresses.

  • Name Translation
  • Domain Lookup
  • Address Resolution
üìñ Learn More

How it works:

You type a domain name (superhero.com) ‚Üí Browser contacts a DNS server ‚Üí DNS server looks up the IP address ‚Üí Returns the IP (e.g., 93.184.216.34) ‚Üí Browser connects to that IP.

Real World: DNS is like a phone book for the internet. Instead of remembering IP addresses, we use memorable domain names.

üîê HTTP/HTTPS

The protocols that power the web with optional encryption.

  • Stateless Communication
  • Request Methods
  • Security (HTTPS)
üìñ Learn More

How it works:

HTTP: Stateless protocol where each request is independent. No built-in security - data sent as plain text.

HTTPS: Same as HTTP but with encryption (SSL/TLS). Protects sensitive data like passwords and payment information.

Status Codes: 200 = OK, 404 = Not Found, 500 = Server Error

üîó Hypertext & Links

The web of interconnected documents that creates the "World Wide Web".

  • Hyperlinking
  • Document Connection
  • Web Navigation
üìñ Learn More

How it works:

Hypertext documents contain links (anchor tags) that reference other resources. Clicking a link navigates you to a new URL. These interconnected documents create the "web."

Example: <a href="superhero.html">Visit Superhero</a>

The Web: Named "World Wide Web" because documents are woven together through hyperlinks.

üéØ URLs

Universal Resource Locators uniquely identify web resources.

  • Format: protocol://domain/path
  • Example: https://superhero.com/batman
  • Components: Protocol, Domain, Path, Query
üéØ Learn More

URL Components:

https://superheroes.com/hero/batman?power=strength

  • Protocol: https (secure) or http
  • Domain: superheroes.com
  • Path: /hero/batman
  • Query String: ?power=strength

HTML, CSS, and other standards ensure compatibility.

  • HTML: Structure
  • CSS: Styling
  • Works: All browsers
üìã Learn More

Web Standards ensure interoperability:

  • HTML: Structures content (elements, semantics)
  • CSS: Styles and layouts the content
  • No JavaScript: Pure HTML and CSS for GitHub Pages compatibility

These standards work across all browsers (Chrome, Firefox, Safari, Edge) and devices (desktop, mobile, tablet).

  • Client: Local processing
  • Server: Database & logic
  • Both: Work together

Client-Side Processing:
